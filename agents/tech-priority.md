# 技术选型优先级

> 本文档为架构师提供技术选型参考。根据 Tech Coach 输出的能力需求优先级（P0/P1/P2），
> 架构师应按照本优先级文档选择最合适的技术方案。

## 一、选型原则

| 优先级 | 选择策略 |
|--------|----------|
| **P0 能力** | 选择**最稳定、最成熟、社区最活跃**的技术。成本最低、风险最低。 |
| **P1 能力** | 选择**功能完善、维护活跃**的技术。平衡功能与复杂度。 |
| **P2 能力** | 选择**轻量、简单、最小依赖**的技术。避免过度工程化。 |

**通用原则：**
- 优先选择团队熟悉的技术
- 优先选择国内企业广泛采用的技术（生态好、招聘容易）
- 优先选择成本最低的方案
- 优先选择零配置或低配置的技术
